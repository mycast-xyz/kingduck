# 기획안 — 게임 메뉴 길이 문제 해결: 즐겨찾기 레일 + 전체게임 플라이아웃

> 작성일: 2026-06-21
> 배경: 게임 수가 늘수록 좌측 아이콘 사이드바(레일)가 세로로 무한정 길어진다. "더보기"로
> 접으면 펼쳤을 때 또 세로로 길어져 근본 해결이 안 됨. → 레일은 **즐겨찾기(또는 기본 3개)만**
> 짧게 유지하고, 전체 게임 목록은 레일 **우측으로 펼쳐지는 가로 그리드 플라이아웃**으로 노출한다.

---

## 0. 현재 상태

| 구분 | 파일 | 동작 |
|------|------|------|
| Desktop 레일 | `src/app/view/agent/desktop/MainMenu.svelte` | 로고 → 캘린더 → **전체 게임 세로 나열**. 즐겨찾기 상단 고정·별 토글은 직전 작업에서 추가됨(`FavoriteService`). |
| Mobile 메뉴 | `src/app/view/agent/mobile/MainMenu.svelte` | 햄버거 → 전체화면 오버레이 드롭다운(세로, 스크롤 가능). 즐겨찾기 상단 고정 적용됨. |
| 즐겨찾기 | `src/app/service/FavoriteService.ts` | `localStorage['favorite_games']` 전역 slug 배열. `toggle(slug)`. 로그인 무관. |

문제는 **Desktop 레일**: 폭 80px 고정인데 게임이 늘면 세로로만 증식 → 하단 테마토글/유저 영역을 밀어내고 스크롤 발생.

---

## 1. 목표 모델

### 1-1. Desktop 레일 (세로, 짧게 고정)

```
┌──────┐
│ 로고 │
├──────┤
│ 📅   │  캘린더(가챠예보)
├──────┤
│ ★게임│  ← 즐겨찾기 아이콘 (없으면 gameList 앞 3개 = 기본 노출)
│ ★게임│
│ ★게임│
├──────┤
│ ▦    │  ← "전체 게임" 트리거 (그리드 아이콘 ri-apps-2-line)
├──────┤
│ 🌙   │  테마토글
│ 👤   │  유저
└──────┘
```

- 레일에 항상 보이는 게임 = **즐겨찾기**. 즐겨찾기 0개면 `gameList.slice(0, 3)`(기본 3개) 노출.
- 그 아래 **전체 게임 트리거 버튼** 하나. 이게 핵심: 레일 세로 길이가 `즐겨찾기 수 + 상수`로 **상한**이 생긴다.

### 1-2. 전체게임 플라이아웃 패널 (가로 그리드)

```
        ┌────────────────────────────────────┐
│ ▦  ●─┤  전체 게임                          │
│      │  ┌────┐ ┌────┐ ┌────┐ ┌────┐        │
│      │  │아이│ │아이│ │아이│ │아이│  ← 2~3열 그리드
│      │  │콘  │ │콘  │ │콘  │ │콘  │        │
│      │  │이름│ │이름★│ │이름│ │이름★│  ← 각 항목 우상단 별 토글
│      │  └────┘ └────┘ └────┘ └────┘        │
│      │  ... (전 게임)                       │
        └────────────────────────────────────┘
```

- 트리거에서 펼쳐지며 레일 **우측에 `absolute left-full top-0`** 배치, 자체 카드 배경 + 그림자.
- 내용: **전체 게임**(즐겨찾기 포함) 그리드. 항목 = 아이콘 + 이름 + 별 토글(추가/해제).
- 가로로 펼치므로 게임이 늘어도 **세로가 아니라 열 수**로 흡수 → 레일을 밀지 않음.
- 카테고리/그룹은 현재 flat 유지(향후 그룹 확장 여지만 남김).

### 1-3. Mobile

- 모바일 오버레이는 이미 전체화면 + 세로 스크롤이라 길이 문제 영향이 작음 → **레이아웃 변경 없음**.
- 즐겨찾기 상단 고정(완료)만 유지. 게임이 더 많아지면 후속으로 그룹/검색 추가 검토.

## 2. 상호작용 결정 (open decisions)

| 항목 | 후보 | 추천 | 비고 |
|------|------|------|------|
| 플라이아웃 열기 | (a) **hover** 열기 / (b) **click** 토글 | **(b) click 토글 + 바깥클릭 닫기** | hover는 실수 노출·터치 미지원. click이 접근성·터치 친화. 트리거 `aria-expanded`. ESC로 닫기. |
| 즐겨찾기 0개 시 | 3개 노출 / 빈 레일 | **gameList 앞 3개 노출** | 사용자 요청. 별 outline 상태 → 누르면 승격. gameList<3이면 있는 만큼. |
| 컴포넌트 분리 | 인라인 / 별도 | **별도 `GameFlyoutPanel.svelte`** | 패널 그리드 분리해 재사용·가독성↑. props: `games`, `url`, 닫기 콜백. |
| 즐겨찾기 정렬 | 추가순 / gameList순 | **gameList 순(현행 유지)** | 현재 `filter`로 gameList 순서 보존. 드래그 정렬은 후속. |

> 위 표의 추천대로 진행하되, hover 선호 시 (a)로 전환 가능(패널에 mouseenter/leave 유지 로직 추가).

## 3. 작업 파일

| 파일 | 변경 |
|------|------|
| `src/app/view/agent/desktop/MainMenu.svelte` | 레일을 `favoritesOrDefault`(즐겨찾기 or 기본3) + 트리거 버튼으로 축소. `flyoutOpen = $state(false)` 상태. 바깥클릭/ESC 닫기. 플라이아웃 패널 렌더. |
| `src/app/view/agent/desktop/GameFlyoutPanel.svelte` (신규) | 전체 게임 가로 그리드 + 별 토글. `data.url` 기반 아이콘. |
| `src/app/service/FavoriteService.ts` | 변경 없음(전역 localStorage 유지). 기본 3개는 컴포넌트 `$derived` 책임. |
| `src/app/view/agent/mobile/MainMenu.svelte` | 변경 없음(이번 범위 외). |

핵심 derived(데스크탑):
```ts
const favoriteGames = $derived(gameList.filter((g) => $favorites.includes(g.slug)));
const railGames = $derived(favoriteGames.length ? favoriteGames : gameList.slice(0, 3));
```

## 4. 엣지/리스크

| 리스크 | 대응 |
|--------|------|
| **z-index 충돌** | 플라이아웃은 레일·콘텐츠 위(z-50+). 레일 hover tooltip과 동시 노출 방지(플라이아웃 열리면 tooltip 억제). |
| **바깥클릭 닫기** | `window` click 리스너 + 패널·트리거 내부 클릭 무시. `browser` 가드. 언마운트 시 해제. |
| **터치(모바일 데스크탑뷰 경계)** | 본 변경은 desktop 컴포넌트 한정. hover 미채택(click)으로 터치 안전. |
| **즐겨찾기 0 → 기본3 혼동** | 기본3은 별 outline로 "즐겨찾기 아님" 시각 구분. 라벨 없이 노출. |
| **접근성** | 트리거 `aria-expanded`/`aria-haspopup`, 패널 `role="menu"`, ESC 닫기, 포커스 트랩은 과하면 생략(메뉴 수준). |
| **반응형 폭** | 플라이아웃 그리드 열 수를 화면 폭에 맞춤(2~3열). 매우 좁은 데스크탑에서 1열 폴백. |

## 5. 단계적 로드맵

| 단계 | 내용 | 완료 기준 |
|------|------|----------|
| **P1 ✅ 완료** | 레일 축소(즐겨찾기/기본3 + 트리거) + `GameFlyoutPanel` 그리드 + click 토글. **바깥클릭/ESC 닫기·aria도 함께 처리됨.** | 레일 길이 상한 생김. 트리거 클릭 시 전체 게임 패널 노출, 게임 이동·별 토글 동작. `pnpm run check` 0 신규 에러. — 충족(시각 검증 완료). |
| **P2 ✅ 완료** | UX 다듬기: 플라이아웃 슬라이드(fly) 애니메이션, 플라이아웃 열림 시 레일 tooltip 억제. (바깥클릭/ESC/aria는 P1에서 완료.) | 키보드/마우스 모두 자연스러움. — 충족. |
| **P3 (진행 중)** | 패널 내 **검색** ✅(자동 포커스·이름 필터·결과없음). 남음: 카테고리 그룹, 관리자 정렬·노출 제어(menuscout D항목, 백엔드), mobile 그룹/검색 | 게임 수 더 늘어도 탐색 용이. |

### 요약

> Desktop 레일은 **즐겨찾기(없으면 기본 3개) + 전체게임 트리거**로 세로 길이에 상한을 두고,
> 전체 목록은 레일 우측 **가로 그리드 플라이아웃**으로 펼친다. 즐겨찾기 인프라(`FavoriteService`)는
> 그대로 재사용. Mobile은 이번 범위 밖(이미 스크롤 오버레이). 권장 상호작용은 **click 토글 + 바깥클릭/ESC 닫기**.
> 신규 파일은 `GameFlyoutPanel.svelte` 하나. P1만으로 길이 문제는 해결되고, P2는 마감 품질, P3는 확장.
