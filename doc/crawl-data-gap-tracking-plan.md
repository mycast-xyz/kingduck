# 기획안 — 크롤 데이터 공백(누락) 추적·알림 (전 게임 공통)

> 작성일: 2026-06-19
> 요청: 신캐라처럼 소스에 아직 없는 섹션(추천/스킬/이미지 등)을 크롤이 **감지해서, 나중에 업데이트해야 함을
> 알려주는** 형태로. 모든 게임 공통 적용. 기획안.

---

## 0. 배경/문제
- 소스(genshin.gg, blablalink, SRS 등)는 **신캐라가 나오면 일정 시차 후** 빌드/이미지/스킬을 채운다.
- 그래서 크롤 시점에 일부 캐릭터는 **추천 광추/유물·이미지·스킬이 비어있음**(예: 에바네시아, 은랑 LV.999).
- 지금은 **조용히 빈 채로** 들어가서, 나중에 소스가 채워졌을 때 **재크롤해야 한다는 걸 사람이 따로 기억**해야 함 → 누락.
- → 크롤이 **"이 캐릭터의 이 데이터가 비었다"를 자동 기록**하고, 어드민이 한눈에 보고 재크롤 타이밍을 알 수 있게.

## 1. 설계 — "기대 vs 실제" 공백 감지 + 기록 + 노출

### 1-A. 감지 (각 크롤러/Sync에서)
- 각 크롤러가 **그 데이터 타입에서 기대되는 필드**를 알고 있다 → 비었으면 gap으로 표시.
  - character: imageUrl 없음 / skills 빈 배열 / (해당 게임이면) recommendedWeapons·lightcones 없음.
  - build: 추천 광추/유물/무기/에코 0건.
- 게임 무관 공통 규칙 + 게임별 기대 필드(작게) 정의.

### 1-B. 기록 (DB) — 2단계
1. **캐릭터 단위 플래그**: `character.metadata.dataGaps = ['image','recommendations','skills']` (또는 별도 컬럼).
   → 프론트/어드민이 캐릭터별로 "무엇이 비었나" 즉시 조회.
2. **크롤 실행 단위 요약**: `crawler_log.metadata.gaps = [{ name, originalId, missing:['recommendations'] }]` + `gapCount`.
   → **이미 만든 크롤러 로그 상세 모달**이 metadata를 그대로 보여주므로, 추가 UI 없이 "이번 크롤에서 N건 누락"이 보인다.

### 1-C. 노출 (어드민)
- **(최소)** 크롤 로그 상세 모달의 metadata.gaps로 확인(이미 구현된 모달 재사용 — 추가 작업 거의 없음).
- **(선택)** 어드민에 "데이터 공백" 뷰: 게임별 `metadata.dataGaps`가 있는 캐릭터 목록 + 무엇이 비었는지 + 마지막 크롤 시각 → "재크롤" 버튼(게임 전체 실행 버튼 재사용). 신캐라 채워질 때 한눈에.
- **(선택)** 크롤 종료 로그에 요약 한 줄: `"[starrail] build: 5 chars missing recommendations (evanescia, 은랑 LV.999, ...)"`.

## 2. 동작 예시
- `starrail/build` 크롤 → 에바네시아·은랑 LV.999는 genshin.gg에 빌드 없음 → 감지 →
  - 그 캐릭 `metadata.dataGaps += 'recommendations'`,
  - `crawler_log.metadata.gaps`에 기록,
  - 종료 로그 "5건 추천 누락".
- 며칠 뒤 genshin.gg가 에바네시아 빌드 게시 → 재크롤 → 추천 채워지면 `dataGaps`에서 'recommendations' 제거.
- 어드민은 공백 뷰/로그로 "아직 비어있는 캐릭터"를 추적.

## 3. 구현 단계
1. **공통 헬퍼**: `markGap(metadata, key)` / `clearGap(metadata, key)` — metadata.dataGaps 갱신(채워지면 자동 해제).
2. 각 크롤러/BuildScraper/Sync에 **기대 필드 검사** 삽입(작게). 채워졌으면 clear, 비었으면 mark.
3. 크롤 종료 시 `crawler_log.metadata.gaps`/`gapCount` 기록(스케줄러·runCrawler 공통 지점).
4. (선택) 어드민 "데이터 공백" 뷰 + 재크롤 버튼.
5. **검증**: starrail build 크롤 → 에바네시아·은랑 LV.999가 gap으로 기록되는지 + 로그 모달 노출 확인.

## 4. 주의
- gap 정의는 **게임/타입별로 다름** → 과하지 않게(핵심 필드만: 이미지/스킬/추천). 하드코딩 분기 최소화, 설정/맵으로.
- "정상적으로 비는 것"(예: 개척자=MC는 추천 없음이 정상) 은 gap에서 제외(화이트리스트). 안 그러면 노이즈.
- 채워지면 **자동 clear**(재크롤로 갱신) — 수동 정리 불필요.
- 전부 크롤러 코드(서버 재현). 별도 외부 의존 없음.

---

### 요약
> 크롤러가 **기대 필드 vs 실제**를 비교해 빈 데이터를 감지 → 캐릭터 `metadata.dataGaps` + 크롤 로그 `metadata.gaps`에 기록 →
> **이미 만든 크롤 로그 상세 모달**(+선택적 어드민 공백 뷰)로 "나중에 업데이트해야 할 캐릭터"를 한눈에. 소스가 채워지면
> 재크롤로 자동 해제. 전 게임 공통, 핵심 필드(이미지/스킬/추천)만, 정상 공백(MC 등)은 화이트리스트로 제외.
