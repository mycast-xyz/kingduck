# 어드민 사용자 관리 시스템 구현 계획

## 1. 개요

어드민 패널 내에서 플랫폼 사용자를 조회, 수정, 제재할 수 있는 관리 기능을 구현합니다.
이는 `admin_feature_plan.md`의 **5. 사용자 관리** 항목에 해당합니다.

## 2. 주요 기능

### 회원 목록 조회

- **필터링**: 닉네임, 이메일, 가입일, 상태(활성/정지/탈퇴)
- **페이징**: 대량의 회원 데이터를 효율적으로 처리
- **요약 정보**: 회원 리스트에서 핵심 정보(닉네임, 등급, 최근 접속일 등) 확인

### 회원 상세 정보 조회 및 수정

- **기본 정보**: 프로필 이미지, 닉네임, 이메일, 소셜 연동 정보
- **활동 이력**: 작성 댓글, 게시글, 좋아요 내역 (추후 구현)
- **권한 관리**: 일반 유저 / 관리자 / 에디터 등 등급 변경
- **상태 관리**: 계정 정지, 활성화, 탈퇴 처리

### 제재 관리 (Ban System)

- **제재 등록**: 사유 및 기간 설정
- **제재 이력**: 과거 제재 내역 조회

## 3. UI/UX 설계

### 페이지 구조

- **목록 페이지 (`/admin/user`)**: 검색 필터와 데이터 테이블로 구성
- **상세 페이지 (`/admin/user/[id]`)**: 탭 형태로 정보 분리 (기본 정보 / 활동 / 제재)

### 메뉴 위치

- 어드민 사이드바의 **'데이터 관리'** 섹션 또는 별도의 **'사용자 관리'** 섹션으로 배치
- 현재 요청에 따라 최상위 메뉴로 추가 예정

## 4. 구현 단계

1. **API 설계 및 문서화** (`admin_user_management.md`)
2. **프론트엔드 UI/UX 구현**
   - 사이드바 메뉴 추가
   - 리스트 페이지 구현
   - 상세 페이지 구현
3. **백엔드 연동**
   - User Service 확장
   - Admin Controller에 User 관련 엔드포인트 추가

## 5. 데이터 모델 (참고)

- `User` 테이블을 기반으로 하며, 관리자 기능을 위해 `UserRole`이나 `UserStatus` 등의 확장이 필요할 수 있음.
