# Kingduck 백엔드 통합 계획

이 계획은 유지보수성, 설정 용이성 및 API 통신 표준화에 중점을 두고 현재 프론트엔드를 "kingduck-server" 백엔드와 통합하는 단계를 설명합니다.

## 사용자 검토 필요

> [!IMPORTANT] > **Base URL 설정**: API 요청 코드를 하드코딩된 `:3000` 대신 환경 변수(`.env`)로 백엔드 URL 설정을 관리하는 것을 제안합니다. 이를 통해 배포 및 개발/운영 환경 전환이 쉬워집니다.

## 변경 제안 사항

### 1. API 설정 중앙화

현재 `axios`는 컴포넌트와 서비스에서 하드코딩된 URL로 직접 사용되고 있습니다.

#### [신규] `src/service/api/client.ts` (위치 변경됨)

- 중앙 집중식 Axios 인스턴스 생성.
- 환경 변수에서 `baseURL` 설정.
- 공통 에러 처리 및 인증 토큰을 위한 요청/응답 인터셉터 추가.

#### [신규] `.env` & `.env.example`

- `PUBLIC_API_BASE_URL` 정의.

### 2. 기존 서비스 리팩토링

기존 서비스를 업데이트하여 중앙 클라이언트를 사용하도록 변경합니다.

#### [수정] `src/routes/+layout.ts`

- 하드코딩된 `http://' + url.hostname + ':3000'` 로직 제거.
- 새로운 API 클라이언트 또는 설정된 Base URL 사용.

#### [수정] `src/app/service/character/CharacterListService.ts`

- 직접적인 `axios` 사용을 중앙 클라이언트로 대체.
- 중앙 클라이언트 사용 시 `currentUrl` 매개변수 제거.

### 3. 타입 정의

백엔드 응답 타입이 명확히 정의되도록 합니다.

#### [신규] `src/lib/types/api.ts`

- 공통 응답 인터페이스 정의 (예: `ResultCode`, `Items`).

## 검증 계획

### 수동 검증

1. **백엔드 시작**: `kingduck-server`가 3000번 포트에서 실행 중인지 확인.
2. **프론트엔드 시작**: `pnpm run dev` 실행.
3. **게임 목록 확인**: 사이드바/메뉴에 전역 게임 목록이 로드되는지 확인 (`+layout.ts` 처리).
4. **캐릭터 목록 확인**: 캐릭터 목록 페이지로 이동하여 데이터 로딩 확인.
5. **콘솔 확인**: CORS 오류나 연결 거부 오류가 없는지 확인.
